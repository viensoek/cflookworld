**GPL 2.0** 是一种 **开源软件许可证（GNU General Public License version 2）**。
它的核心思想是：

> “你可以自由使用、修改、分享软件，但你也必须让别人拥有同样的自由。”

---

## 📘 详细解释

| 项目       | 内容                                                      |
| -------- | ------------------------------------------------------- |
| **全称**   | GNU General Public License, Version 2（GNU 通用公共许可证第 2 版） |
| **发布机构** | 自由软件基金会（Free Software Foundation, FSF）                  |
| **发布时间** | 1991 年                                                  |
| **许可类型** | “Copyleft”（著作权属反向授权）许可                                  |

---

## ⚖️ 它允许你做什么？

你可以自由地：

1. **使用（Use）**
   任何人都可以免费下载、安装、运行。
2. **复制（Copy）**
   可以把源代码或编译好的程序分享给别人。
3. **修改（Modify）**
   可以改代码、加功能。
4. **分发（Distribute）**
   可以公开发布修改后的版本或原版。

---

## 🚫 但有严格条件

这是 GPL 2.0 的重点，也是很多人忽略的部分：

| 条件           | 含义                                          |
| ------------ | ------------------------------------------- |
| **必须公开源代码**  | 如果你发布（分发）了基于 GPL 2.0 的软件或修改版，就必须同时提供完整的源代码。 |
| **继承相同许可证**  | 你发布的修改版必须也使用 **GPL 2.0**，不能改成别的闭源协议。        |
| **不能添加限制条款** | 你不能在别人使用、修改、分发时再加上额外限制（比如收费授权或保密）。          |

这被称为 **“Copyleft（反版权）”** 原则——
👉 任何人都可以自由使用，但必须让后续使用者也保持自由。

---

## 🧠 举个例子

假如你用一个 **GPL 2.0 授权的开源库** 写了一个应用：

* 如果你只自己用，不公开 → ✅ 没问题。
* 如果你把它编译成软件发布给别人用 → ⚠️ 必须附带或提供源代码，并且你的软件也必须使用 GPL 2.0 授权。

---

## 🔄 与其他许可证的区别

| 许可证            | 可闭源发布？ | 必须开源？ | 商业用途？ | 代表项目               |
| -------------- | ------ | ----- | ----- | ------------------ |
| **GPL 2.0**    | ❌ 不可   | ✅ 是   | ✅ 可   | Linux 内核           |
| **MIT**        | ✅ 可    | ❌ 否   | ✅ 可   | React、jQuery       |
| **Apache 2.0** | ✅ 可    | ❌ 否   | ✅ 可   | Android、TensorFlow |
| **BSD**        | ✅ 可    | ❌ 否   | ✅ 可   | FreeBSD、OpenSSH    |

---

## 📜 常见项目使用 GPL 2.0 的例子

* **Linux Kernel**
* **WordPress**（早期 GPL 2.0）
* **MySQL**
* **GCC（GNU 编译器）**

## ✅ 总结

| 项         | 内容                               |
| --------- | -------------------------------- |
| **许可证名称** | GNU GPL v2                       |
| **核心原则**  | 自由使用、修改、分享；强制开源继承                |
| **风险点**   | 商业项目中使用要注意“传染性”条款                |
| **安全用法**  | 自己学习或内部使用可以；若要发布或售卖，请确保符合 GPL 义务 |
